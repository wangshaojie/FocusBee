# 专注力训练营 - 游戏规格文档

## 1. 项目概述

- **项目名称**: 专注力训练营
- **项目类型**: Android 原生应用
- **核心功能**: 包含多个专注力训练小游戏，帮助提升专注力和反应能力
- **目标用户**: 儿童及需要训练专注力的用户

## 2. 技术栈

- 语言: Kotlin
- 最低SDK: 24 (Android 7.0)
- UI框架: Jetpack Compose + XML (混用)
- 架构: MVVM
- 数据存储: DataStore (积分本地持久化)

## 3. 游戏列表

### 3.1 萌音大挑战

**游戏规则**:

- 倒计时 3-2-1 后显示 8 个随机动物
- 必须包含: 狗、猫、羊（容易模仿叫声）
- 点击刷新按钮可重新随机

**动物列表**:

1. 狗 🐕
2. 猫 🐱
3. 羊 🐑
4. 牛 🐮
5. 猪 🐷
6. 鸡 🐔
7. 鸭 🐦
8. 青蛙 🐸

**功能**:

- 倒计时动画
- 动物随机刷新
- 自动刷新功能（默认开启，10秒）
- 背景音乐支持

### 3.2 舒尔特训练 (Schulte Table)

**游戏规则**:

- 在网格中按顺序点击数字 1 到最大
- 训练专注力和视觉搜索能力
- 完成时间越短，专注力越强

**难度级别与关卡**:

| 难度 | 网格大小       | 关卡数 | 总关卡    |
| ---- | -------------- | ------ | --------- |
| 简单 | 3×3 (9个数字)  | 50关   |           |
| 中等 | 4×4 (16个数字) | 50关   |           |
| 困难 | 5×5 (25个数字) | 50关   |           |
| 挑战 | 6×6 (36个数字) | 50关   | **200关** |

**关卡跳转规则**:
- 难度与关卡分离管理，下一关只在同难度内跳转
- 难度完成后显示 "🎉 XX难度已全部通关！" 提示，不再自动跳转

**积分系统**:

| 难度 | 网格 | 基础积分 |
| ---- | ---- | -------- |
| 简单 | 3×3  | 30分     |
| 中等 | 4×4  | 40分     |
| 困难 | 5×5  | 50分     |
| 挑战 | 6×6  | 60分     |

**UI设计**:

- 渐变背景：浅蓝 → 浅紫（柔和护眼）
- 格子样式：白色背景 + 浅蓝边框 + 圆角
- 数字：大字号、高对比度、加粗
- 低难度(3×3, 4×4)：显示可爱emoji图标装饰

**动画效果**:

- 点击缩放：0.92倍，80ms
- 正确点击：变绿色
- 错误点击：红色闪烁 + 轻微抖动
- 网格出现：交错淡入动画
- 奖励弹窗：缩放出现
- 积分飞入：从中心飞向顶部
- 闯关成功背景图切换

**游戏规则**:

- 翻牌配对游戏，训练短期记忆与注意力
- 点击两张相同的卡片即可配对成功

**难度级别与关卡**:

| ---- | -------------- | -------- | -------- |
| 简单 | 3×4 (12张)     | 6对      | 1-50     |
| 中等 | 4×4 (16张)     | 8对      | 51-100   |
| 困难 | 4×5 (20张)     | 10对     | 101-150  |
| 挑战 | 5×6 (30张)     | 15对     | 151-200  |

**关卡跳转规则**:
- 难度与关卡分离管理，下一关只在同难度内跳转
- 难度完成后显示 "🎉 XX难度已全部通关！" 提示，不再自动跳转

**UI设计**:

- 卡片背面：浅蓝色云朵图案
- 卡片正面：显示图标（动物、食物、交通工具等）
- 配对成功：绿色背景 + 缩放动画
- 配对失败：红色闪烁后自动翻转

**动画效果**:

- 卡片翻转：180度3D旋转
- 配对成功：缩放弹跳动画
- 进入游戏：缩放淡入动画

### 3.4 成长系统

**游戏规则**:

- 在网格中找出与众不同的目标
- 随着关卡深入，目标数量增加
- 连续答对获得连击加成

**核心功能**:

#### 星星系统 ⭐

- 完成关卡 → +10 星星
- 连续答对（combo） → 额外奖励
- 计算公式: `星星 = 基础10 + 连击数 * 2`

#### 勋章系统 🏅

| 条件           | 勋章ID          |
| -------------- | --------------- |
| 连续完成3天    | early_bird      |
| 连续完成5关    | focus_master    |
| 单次得分 > 100 | high_score      |
| 连击10次       | combo_king      |
| 完成第一关     | first_win       |
| 解锁所有星球   | planet_explorer |

#### 角色解锁 🎭

| 角色      | 解锁条件 |
| --------- | -------- |
| 狐狸 🦊   | 初始     |
| 猫咪 🐱   | 100星星  |
| 宇航员 🧑‍🚀 | 300星星  |
| 机器人 🤖 | 500星星  |
| 神龙 🐉   | 1000星星 |

#### 星球地图 🌍

| 星球        | 需完成关卡数 |
| ----------- | ------------ |
| 森林星球 🌲 | 0 (初始)     |
| 海洋星球 🌊 | 5            |
| 太空星球 🌙 | 15           |
| 沙漠星球 🏜️ | 30           |

**UI页面**:

1. **HomeScreen**: 星球地图 + 角色选择 + 勋章展示
2. **LevelScreen**: 关卡选择（每个星球5关）
3. **GameScreen**: 小游戏
4. **ResultScreen**: 得分 + 星星奖励 + 勋章获得

**动画效果**:

- 星星增加动画 (+N 飘动)
- 勋章获得弹窗
- 角色/星球解锁提示

实现统一积分系统（Score System）

【一、积分规则】

1. 初始积分为 0

2. 每局游戏结束后，根据星级计算积分：

- 1星 → +1 分
- 2星 → +2 分
- 3星 → +3 分

规则：
points = stars（1~3）

---

【二、数据结构】

定义：

data class GameResult(
val stars: Int, // 1 ~ 3
val duration: Long,
val mistakes: Int
)

---

【三、全局积分管理】

创建 ScoreManager（单例）：

提供方法：

1. addScore(result: GameResult)

逻辑：

- 从 result.stars 计算 points
- totalScore += points

2. getTotalScore()

返回当前总积分

---

【四、存储】

要求：

- 使用 DataStore 或 SharedPreferences
- totalScore 必须持久化
- App 重启后不丢失

---

【五、调用规范（非常重要）】

所有游戏结束时：

必须调用：

ScoreManager.addScore(result)

禁止：

- 在游戏内部直接修改 totalScore
- 在 UI 中直接修改积分
- 每个游戏写一套积分逻辑

---

【六、扩展预留（必须做）】

为未来“积分兑换”预留：

方法：

- spendPoints(amount: Int)
- hasEnoughPoints(amount: Int)

要求：

- 不实现兑换 UI
- 只预留接口

---

【七、校验】

1. stars 必须在 1~3 之间
2. 如果非法，直接修正或报错

---

【八、输出】

请：

1. 实现 ScoreManager
2. 修改现有游戏结束逻辑，统一调用
3. 不要修改 UI 结构
